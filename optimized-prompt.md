## 二、全局视觉风格

整个网站的视觉风格必须统一，请严格遵守以下原则：

### 色彩方案

- **深色主题为默认主题**：背景使用深灰/近黑色（如 `#0a0a0f`、`#111827`），不是纯黑 `#000`
- **主强调色**：使用一种赛博感的蓝/青色（如 `#06b6d4` cyan 或 `#3b82f6` blue），作为按钮、链接、高亮的主色
- **辅助色**：可以用紫色（如 `#8b5cf6`）作为渐变的第二色
- **文字色**：主文字白色/浅灰，次要文字用中灰
- **支持浅色模式切换**：导航栏放一个太阳/月亮切换按钮

### 排版

- 中文字体优先使用系统默认（`"PingFang SC", "Microsoft YaHei", sans-serif"`）
- 英文/代码字体可用 `"JetBrains Mono", "Fira Code", monospace`
- 标题要大而有力，正文大小适中（16px 基准）
- 行间距舒适（1.6-1.8）

### 动效原则

- 页面滚动时各区块有**渐入动画**（fade-in + 轻微上移），使用 Framer Motion 或 CSS
- 按钮 hover 有平滑过渡（0.2-0.3s）
- 卡片 hover 有轻微上浮（translateY -4px ~ -8px）+ 阴影增强
- **不要**：闪烁、抖动、弹跳、大幅位移、荧光色脉冲
- 所有动画都要有 `prefers-reduced-motion` 的降级处理

### 布局

- 最大内容宽度 `max-w-6xl`（约 1152px），居中
- 各区块之间有充足间距（`py-20` 或 `py-24`）
- 移动端（<768px）单栏布局，平板（768-1024px）适当调整，桌面端完整布局
- 导航栏在移动端要有汉堡菜单

## 八、最终强调

### 优先级排序
```
能运行 > 能构建 > 能部署 > 视觉好看 > 功能丰富 > 动效炫酷
```

### 稳定性原则
- 如果某个动画效果可能导致构建失败或运行异常，请选择更简单稳定的实现
- 如果某个依赖可能导致版本冲突，请不要引入
- 宁可少一个花哨效果，也不要多一个运行时报错

### 完整性检查清单
在你输出代码之前，请自行检查：
- [ ] 所有 import 路径是否正确？
- [ ] 所有组件是否都已定义和导出？
- [ ] Tailwind 的 content 配置是否覆盖了所有文件？
- [ ] package.json 的依赖是否完整？
- [ ] vite.config.js 的 base 是否正确？
- [ ] index.html 是否正确引入了入口文件？
- [ ] 所有数据文件是否都已创建？
- [ ] localStorage 的读写逻辑是否正确？
- [ ] 贪吃蛇的游戏循环是否会在组件卸载时清除？
- [ ] 深色/浅色模式切换是否正常工作？
- [ ] 移动端响应式是否正常？
- [ ] GitHub Pages 的路径是否正确？

请现在开始生成完整项目。
